#+title: Doom Remix Bindings

* Implementation decision

Use modern native Emacs keymap APIs instead of =general.el=. Emacs 30 has
=keymap-set= and =defvar-keymap=, which provide most of the readability benefits
without adding another keybinding DSL or package dependency.

These keymaps must work well with =which-key=. As a new Emacs user, I rely on
=which-key= for discovery and memory. Prefix maps should have clear =:doc=
strings, and nested maps should be named by intent, such as files, buffers,
search, notes, and Org clock. After implementing each section, verify that
pressing the prefix shows useful labels in =which-key= instead of anonymous or
confusing keymap names.

Non-leader Evil bindings should use =keymap-set= directly:

#+begin_src emacs-lisp
(keymap-set evil-normal-state-map "j" #'evil-next-visual-line)
(keymap-set evil-visual-state-map "j" #'evil-next-visual-line)
(keymap-set evil-normal-state-map "C-s" #'evil-save)
#+end_src

Leader prefixes should use named keymaps:

#+begin_src emacs-lisp
(defvar-keymap my/leader-files-map
    :doc "File commands."
    "f" #'find-file
    "r" #'recentf-open-files
    "s" #'basic-save-buffer)

(defvar-keymap my/leader-map
    :doc "Global leader keymap."
    "." #'find-file
    "f" my/leader-files-map)

(keymap-set evil-normal-state-map "SPC" my/leader-map)
(keymap-set evil-visual-state-map "SPC" my/leader-map)
(keymap-set evil-motion-state-map "SPC" my/leader-map)
#+end_src

Org localleader bindings should be attached after Org loads:

#+begin_src emacs-lisp
(defvar-keymap my/org-localleader-map
    :doc "Org localleader keymap."
    "t" #'org-todo
    "e" #'org-export-dispatch
    "x" #'org-toggle-checkbox)

(with-eval-after-load 'org
    (keymap-set org-mode-map "\\" my/org-localleader-map))
#+end_src

If many bindings need the same key across several Evil states, add a small local
helper rather than adding =general.el=:

#+begin_src emacs-lisp
(defun my/keymap-set-many (keymaps key command)
    "Bind KEY to COMMAND in each keymap in KEYMAPS."
    (dolist (keymap keymaps)
        (keymap-set keymap key command)))
#+end_src

Keep the initial implementation in =dotfiles/emacs/init.el=. The bottom of the
file will contain a large Evil keybinding section for now; this is acceptable
while the binding design is still changing. If it becomes stable and too large
to navigate, it can be moved later without changing the keymap design.

Place the section after the =evil= and =evil-collection= setup, because it
references maps such as =evil-normal-state-map= and =evil-visual-state-map=.
Use this structure:

#+begin_src emacs-lisp
;;; Evil keybinding helpers

;; my/keymap-set-many and small custom command helpers.

;;; Evil leader keymaps

;; defvar-keymap forms for SPC prefixes.

;;; Evil non-leader bindings

;; j/k, clipboard, save, delete, scrolling, search recentering, and windows.

;;; Evil leader bindings

;; Bind SPC to my/leader-map in normal, visual, and motion states.

;;; Org localleader keymaps

;; defvar-keymap forms for \ prefixes.

;;; Org localleader bindings

;; Attach \ to org-mode-map after Org loads.
#+end_src

* Notes

- =Available= means the function is currently =fboundp= in the running Emacs session.
- =Implemented= means the key binding has been explicitly added to =dotfiles/emacs/init.el=.
- Rows with =my/...= are small custom functions needed to preserve my Neovim habits.
- =SPC= is the global leader.
- =\= is the localleader, replacing Doom's default =SPC m=.
